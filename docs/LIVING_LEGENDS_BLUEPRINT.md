# Living Legends — Blueprint

## Purpose

Living Legends is a narrative system powered by the Structural Coherence Engine (SCE).

It is not a game.
It is not a chatbot.

It is a **living system** where:

- structure governs reality
- AI expresses reality
- the user influences reality

---

## Core Experience

The user does not control characters directly.

Instead, they:

- observe a world unfolding
- feel rising tension or stability
- intervene at key moments
- watch consequences propagate

This creates the feeling of:

> witnessing a living world rather than commanding one

---

## Core Loop

1. World exists in a structural state (SCE)
2. System advances (tick)
3. AI narrates what emerges
4. User is presented with limited interventions
5. User acts or waits
6. System evolves
7. Repeat

---

## System Layers

### 1. Structural Layer (SCE)

Responsible for:
- role conditions
- regime (unity, boundary, fragmentation)
- transitions
- pressure propagation

This layer is:

- deterministic
- consistent
- hidden from the user

---

### 2. World Layer

Defines:
- world type (kingdom)
- role mappings
- resources
- relationships

This layer provides:

- context for interpretation
- structure for AI generation

---

### 3. AI Layer

Responsible for:
- naming characters
- generating bios
- describing relationships
- narrating events

This layer:

- does NOT control system state
- only interprets it

---

### 4. Presentation Layer (UI)

Responsible for:
- chronicle (main feed)
- current state
- suggested interventions

This layer:

- prioritizes clarity
- minimizes noise
- guides attention

---

## Design Principles

### 1. Structure First

All outcomes must originate from SCE.

AI is not allowed to invent outcomes.

---

### 2. Minimal Control

The user is not a commander.

They are:

- a force
- an influence
- a pressure

---

### 3. Meaningful Intervention

Interventions must:

- be limited
- be contextual
- have visible consequences

---

### 4. Emergence Over Script

No fixed story.

Everything emerges from:

- structure
- pressure
- interaction

---

### 5. Readability Over Complexity

The system can be complex internally.

But the user should feel:

- clarity
- flow
- intuition

---

## User Journey

### Phase 1 — World Creation

User selects:
- world type (kingdom)

System generates:
- world name
- cast
- relationships
- initial tone

---

### Phase 2 — Observation

User sees:
- chronicle feed
- world state
- subtle tension signals

No action required.

---

### Phase 3 — Intervention

User is offered:
- 2–4 suggested actions

Examples:
- "Feed the Heir's ambition"
- "Reinforce the King's authority"
- "Let it unfold"

---

### Phase 4 — Consequence

System responds:
- roles shift
- tension rises or falls
- narrative evolves

---

### Phase 5 — Continuation

World continues until:

- collapse
- consolidation
- long-term equilibrium

---

## UI Structure

### Main Focus

Chronicle Feed

- scrollable
- narrative-first
- event-based

---

### Secondary

World State

- roles + conditions
- simple labels

---

### Tertiary

Interventions

- minimal buttons
- context-driven
- not always visible

---

## Interaction Model

The system should feel like:

- reading a living book
- watching a simulation
- influencing a system

NOT like:

- issuing commands
- playing a strategy game
- chatting with an AI

---

## SCE Role in Product

SCE is:

- the invisible engine
- the source of truth
- the generator of meaning

Without SCE:
- this becomes random storytelling

With SCE:
- this becomes structured emergence

---

## AI Role in Product

AI is:

- the storyteller
- the renderer
- the interpreter

AI is NOT:

- the decision maker
- the system controller

---

## V1 Scope

### Included

- kingdom world type
- 4 core roles
- chronicle feed
- suggested interventions
- basic narration

---

### Excluded (for now)

- multiple worlds
- saving/loading
- animations
- multiplayer
- economy systems
- map systems

---

## Success Criteria

The system is working if:

- the world feels alive
- the user feels tension
- interventions feel meaningful
- outcomes feel earned

---

## Long-Term Vision

Living Legends becomes:

- a platform for dynamic worlds
- a demonstration of structural coherence
- a new form of interactive narrative

Eventually:

- multiple world types
- persistent worlds
- visual simulation layers
- game integration

---

## Final Principle

The user should feel:

> "I did not control this world…  
but my presence changed it."
